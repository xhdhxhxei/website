Anti Cheat

Overview

The script works by forcing two types of local scripts (A normal script and a decoy) into PlayerGUI. Then the server sends out a message to both, and the normal script sends a message back to the server. If the server receives the message, the player is not exploiting. If the server does not receive the message, the player is exploiting. If the decoy returns a message, the player is exploiting, we can insure this way that local scripts and remote events are protected, as we never can know what is inside each script. Only the server keeps a registry of every created script there is. The script also regularly replaces and renames random script to make it even more confusing for the exploiter. The script features a settings script, where you can edit how often what gets checked, it connects to the newly added ban API which is used as an IP ban, then logs their ban using webhooks. Its easily expandable to feature databases and websites. There is a saying that all client sided scripts are bypassable, but I really dont know how bypassing this is possible.

Challenges & Solutions

Of course, force spamming scripts and executing them causes some lag, which surprisingly isnt really noticable when used in moderation. I left a few notes in all the script explaining what they do, and their ideal setting. I dont recommend touching anything other than the settings page. As the script is still in testing, Ive added a 'recovery' script, which takes in account server lag, if it becomes too great, the script stops and restarts, clearing all the scripts. Last thing ill note is that spamming a remote event is possible, and makes the whole message system backed up, which in our case doesnt matter as this just bans the player quicker.
